# Smoke Kit

## Цель

- Один короткий тест
- Картинка с хорошим качеством
- Есть референсное изображение
- После теста Pod выключается
- Для видео тестов используем [runpod/MODEL_SETUP.md](/Users/denisvinogradov/Documents/GitHub/[Blender]/runpod/MODEL_SETUP.md)

## Что готовим заранее

1. Workflow JSON
   - один файл под `img2img` или reference-based workflow
2. Reference image
   - маленький тестовый PNG/JPG
3. Source model
   - удобно брать с Civitai
4. Run script
   - создает Pod
   - ждет `ComfyUI`
   - загружает reference
   - отправляет один prompt
   - забирает результат
   - делает `stop`

## Где хранить модель

- Не на локальной машине, если места нет
- На Pod
- Если тестов будет много, на `network volume`
- Для первого дешевого теста `network volume` можно не включать

## Где хранить файлы теста

- `workflow.json` - в репозитории
- `reference.png` - в репозитории
- модели - на Pod

## Самый дешевый порядок

1. Поднять `Pod`
2. Проверить `ComfyUI`
3. Запустить один workflow
4. Сразу `stop`

## Что не делать

- Не держать Pod включенным без дела
- Не качать локально большие модели
- Не делать длинные очереди
- Не тестировать лишние варианты на одном запуске

## Идея автоматизации

- Один локальный скрипт управляет всем
- Сайт нужен только для первого `API key`
- После этого можно жить через терминал
